# 风格接入契约

目前风格为：`wang_yuda`。

具体风格 Skill 位置：由当前 Codex 环境发现可用的风格 Skill；本机固定路径不写入公开仓库。

## 风格研究输入

向风格研究步骤提供：

- 当前日期。
- 当前模拟仓事实源的资金、持仓和委托，事实源规则以 `account-source.md` 为准。
- 通过基础硬限制后的候选池。
- 市场环境、热点、新闻、公告和关键数据证据。
- 长期模拟交易背景：本流程不是一次性选股，而是持续复盘和模拟调仓。
- 复盘与持久化记忆上下文：`trading-memory.md` 摘要、最近操作、观察标的、旧失效条件、后续表现和当前复盘结论。

## 风格 Skill 发现

- 如果当前上下文中已经存在或可以发现对应人物 Skill，优先使用上下文中的人物 Skill。
- 如果当前上下文无法确认对应人物 Skill，必须先显式询问提问人或要求补充本地路径，不得伪造该风格判断。

## 风格输出要求

对每个候选输出：

- `symbol`（股票代码）
- `name`（股票名称）
- `decision`（候选判断）：取值为 `buy`（买入）、`watch`（观察）或 `reject`（剔除）
- `conviction`（判断强度）：取值为 `high`（高）、`medium`（中）或 `low`（低）
- `reason`（判断理由）
- `holding_period`（建议持有周期）
- `sell_conditions`（卖出条件）
- `invalidation_conditions`（失效条件）
- `key_risks`（关键风险）
- `evidence`（证据）

对每个当前持仓输出：

- `symbol`（股票代码）
- `name`（股票名称）
- `position_action`（持仓动作）：取值为 `hold`（持有）、`sell_all`（清仓）、`trim`（减仓）或 `no_action`（不操作）
- `reason`（判断理由）
- `sell_conditions`（卖出条件）
- `invalidation_conditions`（失效条件）

## 规则

- 当前不混合多个风格。
- 除非股票已经在当前模拟仓事实源持仓中，否则不要评估候选池之外的标的。
- 风格判断必须综合历史操作、观察标的、旧失效条件、当前持仓和当前候选，不得每天重新叙事。
- 模拟仓事实源以 `account-source.md` 为准，记忆文件不得替代真实资金、持仓、委托或成交状态。
- 风格判断可以保持主观，但证据和失效条件必须明确。
- 证据不足时，优先选择 `watch` 或 `reject`，不要强行买入。

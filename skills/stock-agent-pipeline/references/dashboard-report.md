# 可视化日报

本文件定义 `$stock-agent-pipeline` 的 HTML 可视化日报规则。HTML 页面是可选的聚合视图，不替代 Markdown 调仓报告、`daily-summary.md` 或模拟仓事实源查询结果。

## 生成时机

- 默认每次运行仍只要求生成 Markdown 调仓报告。
- 收盘后最后一次运行建议生成或更新 HTML 可视化日报。
- 用户明确要求“可视化日报”、“dashboard” 或 “HTML 页面”时，必须生成 HTML 可视化日报。
- HTML 页面只汇总当天信息，不覆盖当天任何 Markdown 原始报告。
- HTML 页面承接详细表格、时间线、持仓明细和可视化复盘；Markdown 调仓报告保持精简，不重复展开所有可视化内容。

## 输出位置

固定保存为：

```text
reports/YYYY-MM-DD/daily-dashboard.html
```

如果目标文件已存在，可以覆盖更新当天 dashboard；不得覆盖当天 Markdown 调仓报告。

## 生成方式

- 第一版由报告整理智能体直接生成单文件 HTML。
- 不新增 Python、Node、数据库、Web 服务或前端框架。
- HTML 使用内联 CSS。
- 默认不依赖外网 CDN、外部 JS 或后端接口。
- 页面内容应适合本地浏览器直接打开。

## 数据来源

HTML 页面可以读取并汇总：

- 当天多份 `reports/YYYY-MM-DD/*-adjustment-report.md`。
- 当天 `reports/YYYY-MM-DD/daily-summary.md`。
- `reports/memory/trading-memory.md` 摘要。
- 本次按 `account-source.md` 查询到的模拟仓事实源状态。

账户资金、持仓、委托和成交只能以 `account-source.md` 定义的模拟仓事实源为准。若事实源状态查询失败，页面必须显示“账户状态不明”，不得用本地报告或记忆文件推断当前账户状态。

风格判断只以 `style-contract.md` 为准。HTML 页面不得硬编码具体风格名称、路径或人物说明。

HTML 页面可以展示当次报告中实际使用的数据源、事实源或风格名称；这些名称只能来自当次报告、`account-source.md` 和 `style-contract.md`，不得在本文件中固定。

## 页面结构

第一版 HTML 包含 8 个区域：

1. 顶部总览：日期、运行次数、是否发生模拟交易、本日一句话结论。
2. 账户状态区：上方使用卡片展示账户级指标，如总资产、可用资金、持仓市值、总仓位、账户收益和账户收益率；下方使用持仓明细表展示个股或港股通等市场粒度信息。
3. 今日操作时间线：盘前计划、开盘确认、尾盘调仓、收盘复盘、临时复查。
4. 持仓复查：当前持仓、原买入理由、今日表现、失效条件是否触发、当前处理。
5. 候选比较表：代码、名称、结论、置信度、核心逻辑、关键证据、主要风险、触发条件、失效条件。
6. 风格判断摘要：当前风格关注点、买入、不买、持有或卖出的核心理由。
7. 记忆更新摘要：写入、保留、压缩或淘汰的记忆。
8. 明日观察计划：重点持仓、重点候选、触发条件、失效条件。

## 安全边界

- HTML 页面不是账户账本。
- 不写入模拟仓事实源密钥、token 或其他认证信息。
- 不把本地报告推断为当前资金、持仓、委托或成交。
- 不使用真实券商、QMT、真实资金交易或港股通交易路径。
- 如果当天没有发生交易，必须明确写明未交易原因。
- 如果发生交易或交易尝试，必须展示交易前后事实源状态或明确标记状态不明。

## 最小样式要求

- 页面应能在宽屏和普通笔记本屏幕上阅读。
- 使用卡片、时间线和表格提升可读性。
- 颜色只用于辅助表达状态，不作为唯一信息来源。
- 重点风险、状态不明、交易失败和失效条件必须醒目。

## 指标口径

- 账户收益率 = 本次展示的账户盈亏金额 / 账户总资产。
- 持仓自身收益率可以展示，但必须明确写成“持仓自身收益率”或放在持仓复查区，不得和账户收益率混用。
- 如果无法确认账户总资产或账户盈亏金额，账户收益率显示“状态不明”。
- 持仓明细表必须把同一持仓的市场、代码、名称、数量、持仓市值、仓位、盈亏和持仓自身收益率放在同一行，不要拆散成多个账户卡片。
- 后续如果展示港股通或其他市场的持仓粒度字段，统一放入持仓明细表，不在账户卡片中另起字段。

# 模拟仓事实源契约

本文件是模拟仓事实源的固定说明位置。其他文档不要分散硬编码具体模拟仓工具名；后续事实源变更以本文件为准。

## 目前事实源配置

- 事实源：`mx-moni`。
- 事实源类型：本地 Skill。
- 调用位置：由当前 Codex 环境发现可用的本地 Skill；本机固定路径不写入公开仓库。
- 当前调用方式：按 `mx-moni` Skill 说明，通过其后端模拟交易接口查询资金、持仓、委托和成交，并执行模拟买卖。
- 未来如果更换事实源，只修改本节配置，不在其他文档分散硬编码。

## 当前规则

- 资金、持仓、委托、成交和模拟交易结果，以本次运行实际采用的数据获取方式为准。
- 当前实际数据获取方式是 `mx-moni` Skill 的后端接口；如未来替换为其他模拟仓 Skill 或用户明确指定的接口，以本文件更新后的配置为准。
- 运行开始前必须确认本次事实源可查询资金、持仓和委托。
- 交易尝试后必须用同一事实源或更稳定的后端查询方式确认资金、持仓、委托和成交状态。
- 如果事实源无法可靠查询账户状态，不交易。
- 如果事实源拒单、状态不明或交易结果无法确认，只记录实际返回和异常，不做本地补单，不伪造成交。

## 记录要求

每次报告必须写明：

- 本次使用的事实源名称。
- 查询方式：按本节当前事实源配置记录；未来更换事实源时按本文件配置记录。
- 交易前资金、持仓和委托状态。
- 如发生交易或交易尝试，交易后资金、持仓、委托和成交确认状态。

## 本地文件边界

- 本地报告和 `reports/memory/trading-memory.md` 只保存 AI 决策记忆。
- 本地文件不是账户事实源，不得用本地报告推断真实资金、持仓、委托或成交。
- 事实源与本地记忆冲突时，以事实源实际查询结果为准。

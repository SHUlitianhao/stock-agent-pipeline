# 复盘与持久化记忆模块

本模块是 `$stock-agent-pipeline` 内部模块，它由主智能体或固定子智能体执行，负责把历史报告整理成本次风格判断可用的决策记忆。

## 记忆边界

- 资金、持仓、委托和成交以 `account-source.md` 定义的模拟仓事实源为准；本文件不指定具体事实源名称、路径或调用方式。
- `reports/memory/trading-memory.md` 是 AI 决策记忆，不是账户账本。
- 不得根据本地报告推断当前真实持仓，不得伪造成交。
- 记忆只记录“当时为什么判断、后来是否兑现、接下来如何处理”。
- 记忆保持精简，只保留重要操作、关键判断、仍需跟踪的标的和可复用经验；重复内容不要反复写。

## 记忆来源

- `reports/memory/trading-memory.md`。
- 最近 5 份 `reports/YYYY-MM-DD/*-adjustment-report.md`。
- 最近 3 份 `reports/YYYY-MM-DD/daily-summary.md`。

如果 `trading-memory.md` 不存在，运行开始时按空记忆处理；本次报告完成后可以创建初始记忆文件。

## 运行开始：生成决策记忆上下文

调用风格 Skill 前，先读取记忆来源并整理为本次决策上下文，包含：

- 最近操作：买入、卖出、减仓、持有、观察、回避。
- 历史理由：当时的核心逻辑、关键证据和主要风险。
- 失效条件：原先设定的失效条件是否触发。
- 后续表现：价格、事件、公告、市场环境或热点变化。
- 当前处理建议：继续持有、继续观察、重新评估、剔除或准备交易。

这一步只生成上下文，不写回记忆文件。

## 运行结束：更新记忆文件

本次报告完成后，基于本次报告和交易后实际查询状态更新 `trading-memory.md`。更新内容包括：

- 历史操作概览。
- 买卖记录摘要。
- 观察标的和关注重点。
- 持仓逻辑和回避理由。
- 兑现或未兑现的复盘。
- 需要长期保留的经验。

如果交易后状态不明，只能记录“状态不明”，不得写成已成交。

连续多次相同结论应合并成当前状态摘要。例如连续“不交易、不追高、继续观察”不逐次展开，只保留最近一次关键证据和当前处理。

## 记忆淘汰

- 最近 5 次运行保留较完整逻辑。
- 更早记录只保留重大买卖、重大判断变化、仍在持仓或仍在观察的标的。
- 已失效且无复盘价值的信息压缩为一句或移除。
- 长期经验只保留可复用结论，不保留冗长叙事。
- 原始候选池、重复市场描述和已无后续价值的观察理由不进入长期记忆。

## 记忆文件建议结构

```markdown
# AI 主观交易记忆

## 当前记忆摘要

概括当前主要持仓逻辑、观察方向、风险和待验证判断。

## 近期操作与复盘

记录最近 5 次运行的重要操作和判断兑现情况。

## 关注标的

记录仍需继续观察的股票、关注理由、触发条件和失效条件。

## 长期经验

记录经过多次复盘仍有价值的判断经验。
```

# 调仓报告模板

每次运行报告保存为 `reports/YYYY-MM-DD/HHmm-adjustment-report.md`。

如果同一分钟发生两次运行，后一次保存为 `reports/YYYY-MM-DD/HHmmss-adjustment-report.md`。

如果同日存在多份报告，可以维护 `reports/YYYY-MM-DD/daily-summary.md` 作为 AI 决策记忆。它只汇总决策和复查重点，不是账户账本，也不得替代 `account-source.md` 定义的模拟仓事实源。

使用以下结构：

```markdown
# YYYY-MM-DD AI 主观选股模拟调仓报告

## 运行摘要

- 模式：查询模式 / 选股研究模式 / 模拟调仓模式
- 运行时段：盘前计划 / 开盘确认 / 尾盘调仓 / 收盘复盘 / 临时复查
- 报告路径：reports/YYYY-MM-DD/HHmm-adjustment-report.md
- 风格：以 `style-contract.md` 为准
- 市场范围：A 股
- 候选数量：
- 新买入目标数量：
- 最终动作：空仓 / 观察 / 持有 / 买入 / 卖出 / 减仓 / 清仓 / 不动
- 模拟交易结果：成功 / 部分成功 / 失败 / 未交易 / 未触发交易

## 交易前模拟仓事实源状态

记录本次事实源名称、查询方式、资金、持仓、委托的关键字段。不要记录或暴露密钥。

## 上次决策复盘

先复盘最近报告或 `trading-memory.md` 中仍有价值的买入、观察、持有和回避标的。

| 上次标的 | 上次结论 | 原理由 | 后续表现 | 原失效条件是否触发 | 当前处理 |
|---|---|---|---|---|---|

## 市场与候选池

说明数据 Skill 来源、热点、候选池生成逻辑、基础硬限制通过情况和最终候选列表。

## 风格判断

说明 `style-contract.md` 中启用风格如何判断 `buy`、`watch`、`reject`，以及当前持仓的 `hold`、`sell_all`、`trim`、`no_action`。

## 候选比较表

用表格呈现候选之间的可比较判断，不要求数字打分，不替代人物 Skill 决策。

| 代码 | 名称 | 结论 | 置信度 | 核心逻辑 | 关键证据 | 主要风险 | 触发条件 | 失效条件 |
|---|---|---|---|---|---|---|---|---|

## 调仓决策

列出每个动作；如果决定空仓、观察或不动，也必须说明原因：

- 股票代码和名称
- 动作：市价买入 / 市价卖出 / 观察 / 持有 / 空仓 / 减仓 / 清仓 / 不动
- 数量
- 理由
- 建议持有周期
- 卖出条件
- 失效条件
- 风险

## 模拟仓事实源执行结果

如果发生模拟交易，记录每笔模拟交易的返回结果。失败、拒单、非交易时间或资金不足必须原样说明。

如果没有发生模拟交易，明确写明未交易原因，例如：风格判断为空仓、候选不满足条件、硬边界拦截、市场环境不支持开仓、用户选择选股研究模式。

## 交易后模拟仓事实源状态

仅在发生模拟交易或交易尝试后记录资金、持仓、委托的关键字段，并说明与交易前相比的变化。

如果没有发生交易，写明模拟仓状态未因本流程发生变化。

## 记忆更新摘要

说明本次写入、保留、压缩或淘汰了哪些记忆。`trading-memory.md` 是 AI 决策记忆，不是账户账本；交易结果必须以 `account-source.md` 定义的模拟仓事实源确认为准。

## 后续观察

列出下一次复查重点、需要补充的数据、以及触发卖出或调整的条件。
```
